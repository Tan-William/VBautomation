  Filtering List Control

1.0 Right-click on the Detail area, then select  Form Header/Footer.


2.0 Right-click on the upper left square button, that would bring out the Form properties sheet as shown on the right.
At the Record Source field, click the dropdown list to select your table/query. For this example, let's select the table called tblCustomers.


3.0 Click on the members of the Fields List, one at a time, drag it to place on the form's Detail section, in the appropriate position you want.


4.0 After you have done, the form in Design View now should look something like this.


5.0 Click on the Combo Box icon, drag to place it in the form's Header, for the purpose of filtering on the contents of the table tblCustomers.


6.0 Choose the second option "I will type in the values that I want"


7.0 Just ignore the 'Number of columns' as you will later code it programmatically. Click Finish.


8.0 Double-click on the combo box Unbound control to bring up the Property dialog box.
Click on the dropdown to select the Label and change it's name to 'Search'; then select Combo1 and change it's name to 'cboSearch'.
Ignore the properties of Row Source Type, Row Source, Column Count and Column Width as we will write the code to take over them.


9.0 Now save Form1 to another name 'frmCustomers', a name that logically make sense. frm denotes as form.


10.0  Open the Form module, follow these 6 steps to program the code for the form's OnOpen event and combo box control's OnClick event.
Close the form module when you are done.


11.0 Open the form frmCustomers and now you will see Customer ID 1125 with all the corresponding details displayed.
Next click on the Search control filter for any Customer ID you like to see, together with all its associated details which
are all the fields that are bound to the table tblCustomers.

 


12.0 Finally, this what the combo box filtering result look likes.
Select the search for, say, Customer ID 1142, you are able to see all its corresponding detail in one page.
